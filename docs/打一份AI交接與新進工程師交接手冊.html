  3. 安裝 Git

    下載安裝後，設定使用者資訊：

    git config --global user.name "你的GitHub使用者名稱" git config --global user.email "你的GitHub信箱"
  4. 取得 API Key

    分別前往 Google AI Studio 和 DeepSeek 平台取得 API Key。

  5. 建立 .env 檔案（重要！）

    複製 .env.template 為 .env，填入實際 API Key：

    # 在 C:\Users\User\ahpal-static 目錄下執行 Copy-Item .env.template .env notepad .env

    .env 檔案內容範例：

    GEMINI_API_KEY=AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
    ⚠️ 重要：.env 不會上傳

    .env 已在 .gitignore 中，不會上傳到 GitHub，也不會部署到 Cloudflare。

    API Key 只存在於您的本地電腦中。

  6. 測試執行
    cd C:\Users\User\ahpal-static .\scripts\ahpal-master.ps1

10. 狀態管理與斷點續傳

10.1 什麼是 article-manifest.json？

article-manifest.json 是 v4.2 新增的狀態追蹤檔，位於 ahpal-static/ 目錄中，記錄每篇文章的生成狀態。

{ "version": "1.0", "articles": { "tech/folding-phone-buying-guide-2026.html": { "hash": "a1b2c3d4e5f6...", "quality": 100, "generated_at": "2026-07-20T00:15:30", "size": 7427 } }, "stats": { "total": 202, "generated": 202, "pending": 0, "failed": 0 }, "last_updated": "2026-07-20T00:25:00" }

10.2 斷點續傳如何運作？

  • 比對哈希值：每次執行時，計算每個檔案的 MD5 哈希值，與 manifest 中的紀錄比對
  • 跳過已生成：若哈希值匹配且檔案大小 ≥ 5KB，則跳過該文章
  • 只生成新增/異常文章：只生成新文章或已損壞的檔案
  • 狀態統計：顯示總計、已生成、待生成、失敗數量

10.3 常用操作

# 查看狀態摘要 cd C:\Users\User\ahpal-static python src/main.py --dry-run # 重置狀態檔（強制重新生成所有文章） python src/main.py --reset # 強制使用 DeepSeek 生成 python src/main.py --force deepseek
✅ 斷點續傳的效益
  • 節省時間：不重複生成已存在的文章
  • 節省 API 成本：只生成真正需要的文章
  • 任務可中斷：中斷後重新執行會從中斷點繼續
  • 品質追蹤：記錄每篇文章的品質分數

11. 常見問題排錯

❌ 401 Unauthorized（Gemini）

原因

Gemini API Key 無效或已過期。

解決方法
  1. 前往 Google AI Studio 取得新 Key（格式：AIzaSy...）
  2. 更新 ahpal-static/.env 中的 GEMINI_API_KEY
  3. 或按 [A] 強制切換到 DeepSeek

❌ 429 RESOURCE_EXHAUSTED（Gemini 配額用完）

原因

Gemini API 免費配額已用完。

解決方法

按 [A] 強制切換到 DeepSeek，或等待次日配額重置。系統已內建容錯切換機制，會自動重試。

❌ Insufficient Balance（DeepSeek 餘額不足）

原因

DeepSeek API 帳戶餘額不足。

解決方法

前往 DeepSeek 平台 充值，¥10 可生成約 1000 篇文章。

❌ .env 檔案不存在

原因

未建立 .env 檔案，或檔案不在正確位置。

解決方法
cd C:\Users\User\ahpal-static Copy-Item .env.template .env notepad .env # 填入實際 API Key

❌ .env 被 Git 追蹤（不應該發生）

原因

.env 不應該被 Git 追蹤，因為包含敏感資訊。

解決方法
cd C:\Users\User\ahpal-static git rm --cached .env echo ".env" >> .gitignore git add .gitignore git commit -m "移除 .env 追蹤"

❌ 文章品質 0 分（字數不足）

原因

AI 生成時因 API 問題產生不完整的內容（< 5KB）。

解決方法
cd C:\Users\User\ahpal-static .\scripts\check-articles.ps1 .\scripts\check-articles.ps1 -Fix .\scripts\ahpal-master.ps1 # 選擇 [4] 只生成文章

❌ Git 推送失敗（rejected）

原因

遠端儲存庫有新的提交，與本地不同步。

解決方法
cd C:\Users\User\ahpal-static git pull --rebase git push

14.1 容錯切換（Failover）

v4.2 新增容錯機制：

  • 異常捕獲：當 Gemini 發生 API 錯誤時，自動切換到 DeepSeek
  • 指數退避重試：失敗後等待 2、4、8 秒後重試

15. 環境變數與 .env 設定

15.1 什麼是 .env？

.env 是環境變數檔案，用於儲存 API Key 等敏感資訊。

⚠️ 重要安全說明
  • ✅ .env 不會上傳到 GitHub（已在 .gitignore 中）
  • ✅ .env 不會部署到 Cloudflare（只部署靜態檔案）
  • ✅ API Key 只存在於您的本地電腦
  • ✅ 其他人看不到您的 API Key

15.2 .env 檔案位置

C:\Users\User\ahpal-static\.env

15.3 .env 檔案內容範例

# Google Gemini API Key（尖峰時段使用） # 請至 https://aistudio.google.com/apikey 取得 GEMINI_API_KEY=AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX # DeepSeek API Key（離峰時段使用） # 請至 https://platform.deepseek.com/ 取得 DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

15.4 如何建立 .env

cd C:\Users\User\ahpal-static Copy-Item .env.template .env notepad .env # 填入實際 API Key 後儲存

15.5 驗證 .env 是否正確

16. 常見問答 (FAQ)

Q1：如何確認 API Key 是否正確？

執行 .\scripts\ahpal-static.ps1 查看環境狀態摘要，會顯示 API Key 格式是否正確。

Q2：文章生成到一半卡住了怎麼辦？

按 Ctrl + C 中斷，檢查 .env 中的 API Key 是否有效。系統支援斷點續傳，重新執行會從中斷點繼續。

Q3：如何只重新生成某幾篇文章？

刪除對應的 HTML 檔案，然後執行 .\scripts\ahpal-master.ps1 選擇 [4]，腳本會自動重新生成被刪除的文章。

Q4：網站更新後沒有看到變化？

清除瀏覽器快取（Ctrl + Shift + R），或檢查 Cloudflare Pages 部署是否成功。

Q5：如何查看目前網站的文章總數？

cd C:\Users\User\ahpal-static .\scripts\check-articles.ps1 .\scripts\check-all.ps1 python src/main.py --dry-run

Q6：.env 檔案需要上傳到 GitHub 嗎？

❌ 絕對不要！ .env 包含 API Key，已在 .gitignore 中排除。請確認 git status 不會顯示 .env。

Q7：DeepSeek 餘額怎麼查？

前往 DeepSeek 平台 → 用量資訊，可看到剩餘餘額和消費金額。

Q8：什麼是 article-manifest.json？

文章狀態追蹤檔，記錄每篇文章的生成狀態，支援斷點續傳。位於 ahpal-static/article-manifest.json。

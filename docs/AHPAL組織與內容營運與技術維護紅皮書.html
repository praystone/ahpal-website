8. 災難復原計畫

8.1 備份策略

  • 每日備份：自動備份至 ahpal-backup/YYYY-MM-DD_HH-MM-SS/
  • 黃金備份：保留最新穩定版本至 ahpal-backup/GOLDEN/
  • 版本控制：所有變更均透過 Git 記錄

8.2 災難復原 SOP

# 情境假設：網站損毀，需從備份還原 # 1. 從黃金備份還原 Copy-Item -Path "C:\Users\User\ahpal-backup\GOLDEN\*" -Destination C:\Users\User\ahpal-static -Recurse -Force # 2. 確認 index.html 正常 Get-Item C:\Users\User\ahpal-static\index.html | Select-Object Name, Length, LastWriteTime # 3. 從備份還原 .env（如果遺失） Copy-Item -Path "C:\Users\User\ahpal-backup\GOLDEN\.env" -Destination C:\Users\User\ahpal-static\.env -Force # 4. 部署到 Cloudflare npx wrangler pages deploy "C:\Users\User\ahpal-static" --project-name=ahpal-pages # 目標：30 分鐘內恢復上線

📜 董事長死命令：文章變更後強制檢查與推送規範

此命令即日起生效，適用於所有工程師與 AI 代理，違者視同違反營運紀律。

🔴 死命令：變更後之強制檢查與推送流程

凡更改或新增任何文章後，必須依序完成以下檢查，確認無誤後方可推送與同步。此流程為強制性，不得跳過或簡化。

📋 強制檢查清單（推送前必做）

階段一：本地檔案完整性檢查

  • 文章數量：python src/main.py --dry-run → 顯示 0 篇待生成
  • 檔案存在：確認所有新增文章皆已生成
  • 檔案大小：≥ 5KB（非空檔案）
  • 首頁更新：新文章標題出現在首頁
  • 分類頁更新：新文章出現在對應分類頁
  • Sitemap 更新：新文章出現在 Sitemap

階段二：內容品質檢查

  • 網頁標題：標題正確顯示
  • 品牌名稱：「雅寶社區」存在
  • AdSense 程式碼：adsbygoogle 存在
  • 返回首頁連結：ahpal.com 連結存在
  • 文章字數：≥ 5KB

階段三：Git 狀態檢查

  • 無未追蹤檔案：git status --porcelain 僅顯示預期變更
  • 無機密檔案：不應出現 .env
  • 無大型檔案：不應出現 audio/（已排除）

🚀 強制執行指令

# 一鍵執行所有檢查 .\scripts\preflight-check.ps1 # 僅檢查 Git 狀態 git status # 僅檢查文章數量 python src/main.py --dry-run

📌 強制紀錄要求

每次推送前，須在提交訊息中註明檢查結果：

git commit -m "新增/修改文章： [文章標題] — 檢查通過（文章數：XXX 篇，品質：OK）"

⚠️ 違反命令之後果

  • 跳過檢查直接推送 → 視同違反營運紀律，需立即修正
  • 推送損壞內容 → 立即從備份還原，並重新執行檢查流程
  • 重複違規 → 將被記錄於技術維護報告中，並追究責任
